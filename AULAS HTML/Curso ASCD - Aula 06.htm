INTRODUÇÃO à SISTEMA OPERACIONAL.


O sistema operacional é um programa ou um conjunto de programas que inicializam o hardware do computador, controla as tarefas que o seu computador realiza e gerencia os recursos do sistema para otimizar o desempenho.

Exemplos de SO:
Windows XP, Windows Vista, Linux Ubuntu, etc.


O sistema operacional deve realizar as seguintes funções:
Inicializar o hardware do computador
Fornecer rotinas básicas para controle de dispositivos
Fornecer gerência, escalonamento  e interação de tarefas
Manter a integridade de sistema


Partes de um Sistema Operacional
Núcleo (Kernel)
Gerenciador de processo
Escalonador
Gerenciador de arquivo

KERNEL

O Kernel é o componente central do sistema operativo da maioria dos computadores; ele serve de ponte entre aplicativos e o processamento real de dados feito a nível de hardware.
 SHELL

A tradução de Shell é concha. O Shell é a camada que envolve o Kernel. Como dito antes o Kernel é o núcleo do Sistema Operacional. É o kernel quem conversa com o Hardware indicando o que deve ser feito.
O Shell é a camada mais acima do Kernel. É a camada que o usuário tem acesso para que ele possa fazer as requisições para o Kernel para que o Kernel entre em contato com o Hardware.

O Shell nada mais é que o interpretador de comandos que transmite ao Kernel o que é para ser realizado.

  ESTRUTURA BÁSICA DE UM SISTEMA

   

   
     
USUÁRIO "Pessoa"
SHELL  Interpreta os comandos o usuario e envia para o Kernel
KERNEL  Recebe os comandos do SHEL e executa no HARDWARE
HARDWARE  Parte física

Exemplo: Deletando um arquivo

O USUÁRIO acessa a mouse na SHELL exeplorer.exe (janelas do Windows) e clica no arquivo para deleta-lo.
O SHELL exeplorer.exe interpreta sua a ação do usuário e trasmite ao KERNEL e o KERNEL vai até o HARDWARE (HD) e elimina o arquivo.




COMO OBTER/PERMITIR ACESSO REMOTO AO SISTEMA
Andre, na estrutura básica de um sistema temos 4 itens: Usuário, Shell, Kernel, Hardware.
Vamos ver qual o melhor item para podermos controlar um sistema:
Hardware: Meio físico ( sem acesso remoto )
Kernel: Linguagem de maquina ( sem possibilidade real de comunicação)
Usuário: acesso ao usuário deixa de ser acesso remoto
Shell : interpreta todos os comandos e permite controlar todo o sistema.

Como vimos obtendo um Shell do sistema podemos ter acesso remoto e controlar todo o sistema.




ABRINDO UMA PORTA DE SEU COMPUTADOR

Quando você abre uma porta de seu computador você permite que outro computador conecte ao seu PC.
Isto não significa que necessariamente o outro computador terá acesso ao seu sistema.
Vamos abrir uma porta de deixa-la apenas em escuta.

Abra seu Insecuritynet 3.X e entre em:
Iniciar -> Gerenciamento Local -> Gerenciar Portas -> Abrir porta para escuta.
Escolha a porta 15

Observe que seu computador esta aguardando uma conexão na porta 15.



 Agora vamos fazer o papel de um computador externo e conectar na porta 15 utilizando apenas o Prompt de comandos.

 Abra o Prompt de Comando e digite:
 telnet <seu ip> 15
 Exemplo:
 telnet 189.7.37.216 15
 
 

  Tecle Enter
 
   

    Observe que existe a conexão porem não aparece nada, digite qualquer coisa.

     Todos os dados que você digitar serão enviados para a porta 15 do "computador remoto" (no caso você mesmo)

     A porta 15 não está direcionada para nenhum programa, apenas está recebendo os dados.
     A tela continua "preta" porque não existem nada ligado à porta 15 e por isto nada é enviado.

     para visualizar os dados enviados para a porta 15 visualize a tela de escuta do Insecuritynet 3.X <listng 15>

 

   Neste exemplo voce abriu a porta 15 de seu PC e direcionou os dados para a tela de visualização do Insecuritynet 3.X
   
   
    Nesta conexão não é possível acessar o sistema do PC 2.
    Os dados são apenas enviados para a tela de escuta do Insecuritynet, e não para uma SHELL do Sistema.




ABRINDO UMA PORTA PARA UMA SHELL
   
(Cuidado ao realizar esta experiência, pois durante a mesma seu computador ficará inteiramente vulnerável)

Andre, agora vamos abrir uma porta para a SHELL CMD.exe
Fechea janela de escuta do Insecuritynet  
 Abra seu Insecuritynet 3.X e entre em:
Iniciar -> Gerenciamento Local -> Gerenciar Portas -> Abrir porta para cmd.exe.
Escolha a porta 15

Observe que seu computador esta aguardando uma conexão na porta 15.



Abra o Prompt de comandos novamente e conecte via telnet.

 

  Tecle Enter
 
 
 
Observe que agora temos total acesso ao computador pela porta 15, é como se você estivesse do Prompt de comando do computador remoto.



Todos os dados enviados para a porta 15 está sendo direcionado para a SHELL cmd.exe.
 Como vimos anteriormente o SHELL permite que você controle o Sistema.



PARA FECHAR A PORTA DE CONEXÃO BASTA FECHAR A JANELA "LISTEN" do INSECURITYNET
O Insecuritynet não manterá sua porta aberta.




Conclusão

Para obter acesso remoto à um computador não basta apenas uma porta aberta, mas esta porta precisa estar direcionada a uma SHELL (programa que permita que voce interaja com o sistema, neste caso o Prompt de comando CMD.exe)
