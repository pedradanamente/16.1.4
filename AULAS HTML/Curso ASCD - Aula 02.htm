Endereço IP


O uso de computadores em rede e, claro, a internet, requer que cada máquina tenha um identificador que a diferencie das demais. Para isso, é necessário, entre outras coisas, que cada computador tenha um endereço, alguma forma de ser encontrado.


 O uso do protocolo TCP/IP não é completo se um endereço IP não for utilizado. Se, por exemplo, dados são enviados de um computador para outro, o primeiro precisa saber o endereço IP do destinatário e este precisa saber o IP do emissor, caso a comunicação exija uma resposta. Sem o endereço IP, os computadores não conseguem ser localizados em uma rede, e isso se aplica à própria internet, já que ela funciona como uma "grande rede".

 
O IP (na versão IPV4) é composto por 32 bits dividido em quatro grupos de 8 bits.
Antes de continuar com o IP vamos entender o que é BIT.

Bit = É a menor unidade de informação que pode ser armazenada ou transmitida. Exemplo: 0101 (4 bits)
Byte = conjunto de 8 bits.

Com apenas um BIT (podemos gerar duas informação) 0 ou 1  (2 X 1)
Com dois BITs (podemos gerar 4 informações) 00  ,  01  , 10 , 11  (2 X 2)
Com  três BITs (podemos gerar 8 informações) (2 X 2 X 2)
Com  quatro BITs (podemos gerar 16 informações) (2 X 2 X 2 X 2)
.
.
.
Com  oito BITs (podemos gerar 256 informações) (2 X 2 X 2 X 2 X 2 X 2 X 2 X 2)

 
Voltando ao IP, o endereço IP é formado por quatro grupos de 8 BITs, portanto em cada grupo podemos ter 256 números diferente, como temos o número 0 cada grupo irá de 0 à 255.
Como são 4 grupos temos (256 X 256 X 256 X 256 = 4.294.967.296) possíveis endereços.
Podemos concluir que toda a faixa de IP vai de 0.0.0.0 à 255.255.255.255

 



FAIXA DE IP (IP Range)

 
Faixa de IP é um determinado grupo de IPs que estão entre Dois IPs Definidos.

 
Por exemplo:

A faixa de IP entre 127.0.0.1 e 127.0.0.7:
127.0.0.1
127.0.0.2
127.0.0.3
127.0.0.4
127.0.0.5
127.0.0.6
127.0.0.7

Outro exemplo (Lembrando que 255 é o ultimo numero do digito)
A faixa de IP entre 127.0.0.253 a 127.0.1.3:
127.0.0.253
127.0.0.254
127.0.0.255
127.0.1.0
127.0.1.1
127.0.1.2
127.0.1.3

Temos 4.294.967.296 possibilidades de IPs diferente, tudo isto precisa ser organizado e regulamentado, para isto temos o seguinte:

 



 DIFERENÇAS DE APLICAÇÃO DE IP PARA REDE LOCAL E PARA INTERNET.

Andre, o endereço IP pode ser usado para identificar um computador dentro de uma rede local (privada) ou para identificar um computador na internet
Em uma rede local na sua casa seu computador pode ter o IP: 192.168.0.4 (por exemplo) e em outra rede local na casa do seu amigo o PC dele também poderá utilizar este mesmo endereço.

 
SUA CASA CASA DE SEU AMIGO
           
                       

 

Já o IP utilizado para localizar um computador na internet (189.25.4.8 e 201.45.8.79) não pode existir dois IPs iguais.

 



 CLASSES DE IP PARA REDE LOCAL:

Como os endereços IP usados em redes locais são semelhantes aos IPs da internet, usa-se um padrão conhecido como IANA (Internet Assigned Numbers Authority) para a distribuição de endereços nestas redes. Assim, determinadas faixas de IP são usadas para redes locais, enquanto que outras são usadas na internet. Como uma rede local em um prédio não se comunica a uma rede local em outro lugar (a não ser que ambas sejam interconectadas) não há problemas de um mesmo endereço IP ser utilizado nas duas redes. Já na internet, isso não pode acontecer. Nela, cada computador precisa de um IP exclusivo.

O padrão IANA divide a utilização de IPs para redes em, basicamente, 3 classes principais e duas que podem ser consideradas secundárias. Esta divisão foi feita de forma a evitar ao máximo o desperdício de endereços IPs que podem ser utilizados em uma rede:
Classe A: 1.0.0.0 até 126.0.0.0 - Permite até 16.777.216 de computadores em cada rede (máximo de 126 redes);
Classe B: 128.0.0.0 até 191.255.0.0 - Permite até 65.536 computadores em uma rede (máximo de 16.384 redes);
Classe C: 192.0.0.0 até 223.255.255.254 - Permite até 256 computadores em uma rede (máximo de 2.097.150 redes);
Classe D: 224.0.0.0 até 239.255.255.255 - multicast
Classe E: 240.0.0.0 até 255.255.255.255 multicast reservado





IP ESTÁTICO E IP DINÂMICO:

Estático: é o IP Fixo que não muda (exceto seja feita a mudança manualmente).

Dinâmico: o IP muda a cada conexão (por motivos de segurança)


IP NAT

É possível conectar vários computadores a Internet utilizando apenas 1 endereço IP de internet. Basta você conectar um computador à intenet e usá-lo como servidor de internet. Os outros computadores você conecta à este Servidor.
Neste caso estamos criando um Rede Privada onde teremos que atribuir endereços IPs para estes computadores, no entanto estes IPs não pertencem à internet mas apenas à sua rede local.

Desvantagens: Os computadores conectados ao servidor não possuem um endereço na internet, portanto não podem ser usados como servidores (exceto com redirecionamento de portas, como veremos no decorrer do curso).

Vantagens: Com apenas um IP disponível na Internet você conecta vários computadores à internet.
Cada computador não possui um endereço na internet, isto o torna mais seguro.

Para descobrir o seu IP basta utilizar o ipconfig (presente no Windows).
Abra o prompt de comando e digite: ipconfig

COMO DESCOBRIR O IP DE UM DETERMINADO SERVIDOR.

Para descobrir o IP de um servidor basta utilizar o ping (presente no Windows).
Por exemplo, para descobrir o IP do servidor www.google.com
Abra o prompt de comando e digite: ping www.google.com
Observe que o ping retorna com o IP: 66.249.90.104

Andre, o Insecuritynet 3.X possui uma ferramenta que retorna o IP qualquer servidor na internet.
Inicio -> Scan e Monitoramento -> Captura de IP -> Capturar IP de um determinado endereço DNS

Como estes sites funcionam se a maioria dos IPs não são fixos são dinâmicos (mudam a cada conexão)?
R: O site possui um banco de dados com os IPS fixos do país ou do mundo e seu respectivo CEP.
Os IPs dinâmicos próximos aos IPs fixos estarão também próximos na faixa de IP distribuída.

Como localizar o usuário de um determinado IP?

R: Neste caso estes dados por motivos de segurança não são públicos, só podem ser obtidos com autorização judicial.
Por exemplo: Você quer saber quem estava utilizando o IP: 189.35.151.28  no Dia 06/10/2010 as 15:00
Este Ip pertence à uma faixa de IP distribuída à um provedor, é possível localizar o provedor, mas somente o provedor deste IP tem os dados de hora data e assinante da banda de internet (fica tudo registrado em Logs). O provedor apenas irá liberar estes dados sob ordem judicial.

 



Escondendo seu IP.

 
Você pode esconder seu IP utilizando proxys, a idéia é muito simples, você conecta em outro computador e o outro computador navega para você. Assim o IP que ficará nos logs é o IP do computador a qual você esta conectado.

Existem muitos servidores Proxys que oferecens este serviços:

Exemplo:
http://www.zend2.com/

 
Exemplo, quando você entra no www.hostip.info seu IP automaticamente é localizado.
Experimente entrar no ipgeo.com utilizando o http://www.zend2.com/
Ele irá detectar o endereço da Proxy e não o seu:

COMO SABER SE O ENDEREÇO IP DE SEU COMPUTADOR É VÁLIDO NA INTERNET OU É UM IP NAT.

Compare o endereço mostrado no ipconfig com o endereço mostrado no site http://meuip.datahouse.com.br/

Observe que neste caso, os dois endereços são iguais, portanto o IP de meu computador está válido na internet:
ipconfig (prompt) Site MEUIP

Geralmente IPs que começam com 192 ,  168  e  10  são IPs NAT e não são válidos na internet.
Geralmente IPs Validos da Internet (Brasil) começam com 189 ,  200  , 201 , 187
(isto não é uma regra, pois existem faixas de IPs definidas e liberdade para configuração de endereço de IP em redes locais)




LOCALHOST

O IP Localhost por padrão é: 127.0.0.1
O IP Localhost não é válido na internet e também não é válido em sua rede local. Então para que ele serve?
 O localhost é usado para a comunicação com a própria máquina.
Ser capaz de se comunicar com a máquina atual como se estivesse se comunicando com uma máquina remota é útil para a finalidade de testes, assim como para usar recursos localizados na máquina atual, mas que se esperariam serem remotos. Por exemplo:
Se você tiver um Servidor HTTP instalado em seu computador, você pode testá-lo utilizando o endereço:
http://127.0.0.1




Obtendo um novo endereço IP (apenas para IPs Dinâmicos)

Importante:
Esta ferramenta irá funcionar apenas se seu computador estiver ligado diretamente ao modem, sem nenhum roteador.

Caso você possua IP dinâmico, à cada conexão você recebe um novo endereço.
Para obter um novo endereço bastaria voce simplesmente desconectar da Internet e conectar novamente. Mas você não precisa reiniciar seu PC ou Modem para isto.
Você pode utilizar os comandos no Promtp:
ipconfig /release
ipconfig /renew

Ou simplesmente Utilizar a ferramenta especifíca para isto no Insecuritynet 3.X
Inicio -> Gerenciamento Local -> Obter novo IP
