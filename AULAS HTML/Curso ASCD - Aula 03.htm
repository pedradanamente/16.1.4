PROTOCOLO TCP 


Antes de iniciarmos os estudos sobre TCP, vamos voltar na pilha TCP/IP.
IP: Protocolo responsável pelo endereçamento de rede
TCP: Protocolo responsável pelo transporte dos dados.

Exemplo:
Quando você envia um arquivo para outro computador você necessita dos endereços das duas máquinas (protocolo IP) e também precisa de uma forma de transmitir estes dados de modo que as duas maquinas consigam enviar e receber o dados (protocolo TCP)

 O TCP (Transmission Control Protocol - Protocolo de Controle de Transmissão), é um protocolo orientado a conexões confiável que permite a entrega sem erros de um fluxo de bytes.

 Funcionamento do TCP:

 
Como é feita a conexão:

1-) Computador cliente solicita um “pedido de conexão” ao computador servidor

2-) O computador servidor verifica o pedido e retorna com a “permissão de conexão”

3-) O computador cliente  recebe as informações de confirmação enviadas pelo servidor e envia estas confirmações de volta ao servidor. O servidor recebe as informações, verifica que elas estão corretas e estabelece a sessão.

 
Após a sessão estabelecida (RESUMO):

O remetente envia os dados ao destinatário.

O destinatário confirma de se os dados chegaram corretamente e solicita um novo pacote de dados ou “pede” para enviar novamente porque os dados foram corrompidos.

Assim que todos os dados foram enviados a sessão é encerrada.

 



ANÁLISE DO PROTOCOLO TCP

CONEXÃO:

Cliente: SYN (solicita a abertura da conexão)
Servidor: SYN (avisa que a porta está disponível)
Servidor: ACK (confirma o recebimento)
Cliente: ACK (confirma e inicia a conexão)


TRANSMISSÃO DO DADO:
Cliente: DATA (é enviado o pacote com o dado)
Servidor: OK (a confirmação, depois de verificar a integridade do pacote)

 

TERMINO

Cliente: FIN (solicita o fechamento da conexão)
Servidor: ACK (confirmar o recebimento do FIN para encerrar a sessão)
Servidor: FIN (encerra a sessão do servidor para o cliente)
Cliente: ACK (confirma o FIN do servidor)


 

As principais características do protocolo TCP são as seguintes:

    * TCP permite entregar ordenadamente os datagramas provenientes do protocolo IP
    * TCP permite verificar a onda de dados para evitar uma saturação da rede
    * TCP permite formatar os dados em segmentos de comprimento variável a fim de os "entregar" ao protocolo IP
    * TCP permite multiplexar os dados, quer dizer, fazer circular simultaneamente informações que provêm de fontes (aplicações, por exemplo) distintas numa mesma linha

 



ANÁLISE DIRETA DO PROTOCOLO TCP:

 
 

 
PORTA DE ORIGEM: Ponto  terminal  da conexão da origem(16 BTIs)
PORTA DE DESTINO: Ponto  terminal da conexão da destino (16 BTIs)
Número de sequência: Identifica a posição do segmento no fluxo de dados (32 BITs)
Número de Confirmação: Confirma o recebimento da dado recebido anteriormente e indica o próximo segmento aguardado.
Tamanho do cabeçalho TCP: (32 BITs)
Checksum: Verificação de erros

 



  PORTAS TCP:

 

Andre, observe que no protocolo TCP temos um espaço de 16 BTIs para Portas de origem e 16 BTIs para a porta de destino.

Logo o numero máximo de portas é:
= 2X2X2X2X2X2X2X2X2X2X2X2X2X2X2X2X = 65.536 portas.


PARA QUE SERVE AS PORTAS:

Imagine uma empresa com vários setores e um único número de telefone, e precisa ter acessos diretos aos setores de forma externa. A solução seria usar um PABX com vários ramais. Exemplo:
Ramal 01 = Recepção
Ramal 02 = Diretoria
Ramal 03 = Sala de reunião
Ramal 04 = Controle de estoque

No computador é a mesma coisa, existe apenas um endereço IP, mas vários programas e serviços, por exemplo: Navegador, Messenger, Jogos online, etc.). Cada serviço deve ter sua porta específica assim com apenas um endereço IP você pode acessar diferentes serviços do computador.


 

 

As portas TCP mais usadas (também chamadas de "well known ports") são as portas de 0 a 1023, que são reservadas para serviços mais conhecidos e utilizados, como servidores web, FTP, servidores de e-mail, compartilhamento de arquivos, etc. A porta 80, por exemplo, é reservada para uso de servidores web, enquanto a porta 21 é a porta padrão para servidores FTP. A porta "0" é reservada, por isso não entra realmente na lista.

 



PROTOCOLO UDP

 
Não vamos nos aprofundar no protocolo UDP, mas é importante que você saiba que o protocolo UDP é muito semelhante ao TCP porem não existe confirmação se os dados chegaram corretamente.

 
PORTAS UDP:

O sistema é parecido com o TCP, com exatamente o mesmo numero de portas.

Segue abaixo algumas portas padrões UDP:

nomes de domínios)

DIFERENÇAS ENTRE UDP E TCP.

A grande diferença entre os dois protocolos de transporte é que o protocolo TCP é confiável porque checa se todos os dados foram entregues corretamente, caso o dado seja corrompido durante a transmissão o protocolo TCP rejeita o dado e solicita um novo envio. Já no protocolo UDP isto não acontece, um dado corrompido é entregue de forma corrompida.
O Protocolo UDP, não estabelece uma conexão antes de envia os dados (Ao contrario do TCP). Se o destinatário está Online ou não, ele simplesmente envia os dados, sem saber se vão chega ao destino ou não.

 Mas porque usar UDP?

Como os dados não são conferidos a velocidade de transmissão é significativamente maior.  Em serviços te transmissão de VOZ, por exemplo, você necessita de alta velocidade de transmissão e caso algum BYTE seja corrompido não implicará de forma significativa no dado entregue.
