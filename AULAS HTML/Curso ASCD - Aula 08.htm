CONEXÕES
 

Para existir uma conexão entre dois computadores é necessario:

 Computador Cliente
Programa Cliente
Computador Servidor
Programa Servidor
Endereços IP
Portas definidas
Meio Físico
Protocolo

Exemplo:



 
Computador Cliente: Seu Computador
Programa Cliente: Navegador (Firefox)
Computador Servidor: Servidor Web
Programa Servidor: Servidor HTTP (Apache)
Endereços IP: 189.25.1.4 e 201.58.4.7
Portas Definidas: 1542 e 80
Meio Físico: Cabo, Fibra óptica, onda de radio, etc.
Protocolo: TCP/IP




ESTADOS DE UM CONEXÃO TCP

 Para uma conexão ser estabelecida é necessário uma série de processos.
 Durante estes processos definimos vários estados para uma conexão.

Regra entre cliente e servidor:

O computador servidor fica aguardando uma conexão
O computador cliente é quem solicita a conexão.


Andre, quando o computador servidor quando está aguardando uma conexão ele está no estado LISTEN.

Quando o cliente solicita uma conexão é enviado o pacote SYN.
Neste momento o cliente que acabou de enviar o pacote está no estado SYN-SENT

O Servidor recebe o pacote e envia outro pacote SYN ACK  respondendo a solicitação, neste momento o servidor está no estado SYN-RCVD.

 O servidor confirma a conexão com um pacote ACK e é iniciada a conexão, neste momento o estado da conexão entre cliente e servidor é ESTABLISHED.



 

Com o decorrer da conexão pode acontecer vários fatores que irá determinar vários outros estados.

     * FIN-WAIT-1: O estado que um host se encontra após ter enviado um pacote FIN inicial pedindo um fechamento correto da conexão TCP.
    * CLOSE-WAIT: O estado da conexão do host que recebeu um FIN inicial e envia de volta um ACK para confirmar o FIN.
    * FIN-WAIT-2: O estado da conexão do host que recebeu a resposta ACK para seu FIN inicial, e indica que agora está esperando um FIN final.
    * LAST-ACK: Esse estado indica que o host acabou de enviar seu segundo FIN, que é necessário para encerramento correto da conexão TCP, e está aguardando uma confirmação.
    * TIME-WAIT: Nesse estado encontra-se o host iniciador que recebeu um FIN final e enviou um ACK para fechar a conexão. Nesse momento ele não irá mais receber nenhuma confirmação do ACK que acabou de enviar, portanto espera um período de tempo para fechar a conexão.
    * CLOSED: pode-se considerar como “sem estado”. Esse estado existe antes que uma conexão seja iniciada ou quando ela é finalizada.

  CONEXÕES EM SEU COMPUTADOR

   Andre, um simples ato de visualizar um website gera varias conexões. alem da conexão direta com o servidor do Site também você poderá estar conectando em outros servidores que hospeda Banner, Iframes, etc.
   Neste caso (Visualizando um webSite) seu computador é um Cliente.
   
   Se voce hospeda algum servidor de JOGO, você fica aguardando uma conexão externa, neste caso seu computador esta sendo um Servidor.

   Portanto seu computador pode agir como cliente e servidor, dependendo de cada situação.

   Para seu computador ser um servidor e acessível aos clientes é necessario:
   
   Endereço externo definido (IP ou DNS)
   Porta Definida
   O IP deve ser válido na Internet, caso contrario somente computadores de sua rede poderão acessar seu servidor.



   
Situações de conectividade
 

 Veja esta situação onde não temos redes privadas
 


   Neste exemplo o PC-A pode iniciar uma conexão com o PC-B, pois o PC-B está visível para o PC-A.
    PC-A (inicia a conexão) = Cliente
    PB-B (recebe a conexão) = Servidor


Agora com o computador cliente dentro de uma rede privada



   Neste exemplo o PC-A pode iniciar uma conexão com o PC-B, pois o PC-B está visível para o PC-A.
   O PC-A (192.168.0.2) irá conectar no modem e o modem repassa a conexão para o PC-B
    PC-A (inicia a conexão) = Cliente
    PB-B (recebe a conexão) = Servidor

Agora com o servidor dentro de uma rede privada.



Agora temos um problema para o PC-A conectar no PC-B.
O PC-A consegue enxergar até o modem (189.200.5.9), como seria possível direcionar a conexão para o PC-B (192.168.0.2)?
A resposta para esta pergunta é: Redirecionado portas no modem para um computador específico.

Para o PC-A iniciar a conexão com o PC-B é necessário utilizar a definicão de porta, por exemplo:
Vamos imaginar que o PC-B seja um servidor FTP (porta 21)

O Modem deve ser configurado para direcionar os pacotes recebidos na porta 21 para o computador 192.168.0.2 (PC-B)



Do ponto de vista do PC-A (Cliente) o servidor será o modem (189.200.5.9)
Nao existe padrão para a configuração do modem, cada fabricante tem sua própria maneira e manual de instruções.

Agora vamos a uma outra situação, quando tanto o cliente como o servidor estão em uma rede privada



Se o modem do servidor  (PC-B) estiver configurado para redirecionar a porta da conexão o PC-A pode iniciar a conexão normalmente com o PC-B

Agora veja este exemplo, onde tanto o cliente como o servidor estão em uma rede privada  e não é possível redirecionar as portas no modem.  A solução será um servidor externo para ligar o cliente ao servidor.



Desta forma tanto o PC-A como o PC-B conectam no servidor externo e este servidor faz a conexão entre as duas máquinas, é como acontece um um bate-papo no Facebook.
