Sobre a questão 1 da aula 09
Se tratando de conexão reversa, qual a afirmação verdadeira?
Resposta correta:O Computador que inicia a conexão fica vulnerável ao computador que recebe a conexão.


"O computador que inicia a conexão é um servidor"
Incorreto porque um servidor fica aguardando uma conexão.

"O computador que recebe a conexão possui o mesmo IP que o coputador cliente"
Incorreto porque dois computadores não podem possuir o mesmo IP em uma conexão

"O Computador que recebe a conexão fica vulnerável ao computador que inicia a conexão"
Incorreto porque o computador que recebe a conexão é o computador que vai ter acesso ao outro.



Utilizando conexão reversa para permitir acesso ao seu computador


Conexão reversa não é  apenas utilizada em Backdoors, podemos fornecer acesso ao nosso computador “entregando” a shell CMD.exe em uma conexão.

Observe a situação:
Você está dentro de uma rede privada, seu IP é NAT (não válido na internet) e você deseja permitir acesso via linha de comando em seu computador para um outro computador que possua um IP válido na Internet.



 

Note que o computador que irá acessar o seu não poderia iniciar a conexão, pois seu IP (192.168.0.2) não é válido na internet. Mas seu computador pode iniciar uma conexão.
Neste caso, o computador que irá acessar o seu deve deixar uma porta em escuta e você deve iniciar a conexão “entregando” a shell de seu sistema.

Vamos fazer na prática, mas usando seu próprio computador para enviar e receber a shell de seu sistema.
Iremos utilizar o Insecuritynet 3.X porque o Windows não possui nenhuma ferramenta que permita fazer a experiência seguinte.

O primeiro passo é definir a porta do computador que irá receber a shell  “cmd.exe”.
Vamos utilizar a porta 32

Agora vamos deixar a porta 32 em estado de escuta:


Abra o Insecuritynet 3.X e entre em:
Iniciar -> Gerenciamento Local -> Gerenciar Portas -> Abrir porta para escuta.
Escolha a porta 32

Veja que a porta 32 está aberta a aguardando uma conexão
 


Agora vamos simular que outra pessoa queira nos entregar o acesso a shell cmd.exe utilizando o Insecuritynet 3.X

(Não feche a janela de escuta da porta 32)


Abra o Insecuritynet 3.X e entre em:
Iniciar -> Gerenciar conexões -> Criar conexões -> Enviar Processo (modo conectar)

 

Digite o IP da pessoa que irá receber (neste caso o mesmo computador) e a porta definida anteriormente (32)



Escolha o processo que você deseja entregar (no caso cmd.exe)




Veja agora a janela onde a porta 32 se encontrava em estado de escuta

 

Temos total acesso ao sistema.
