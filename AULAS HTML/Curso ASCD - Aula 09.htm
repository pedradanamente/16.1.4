CONEXAO REVERSA
 


 
 Conexão reversa é uma técnica utilizada para burlar sistemas de defesa, "enganando" o firewall e obtendo acesso ao sistema.

   Andre, o conceito é simples:
   
  O atacante cria um Backdoor CLIENTE e faz com que seu computador vire servidor deste Backdoor.
  Este backdoor deve contar o endereço do atacante e a porta definida do servidor (Computador do Atacante)
 
 O atacante envia o backdoor para o alvo e assim que o backdoor for executado o computador alvo se conecta no computador do atacante. Porem o programa utilizado no backdoor permite apenas que o atacante tenha acesso a SHELL do Alvo.

  Características de uma conexão reversa:

  •  O atacante não precisa do endereço do alvo
  •  O backdoor não abre uma porta de escuta no computador alvo
  •  O computador alvo não age como um servidor, mas sim o computador do atacante.
  • O alvo pode estar dentro de uma rede privada que mesmo assim haverá conexão


   REQUISIÇÕES PARA O ATACANTE:

   O atacante deve possuir um endereço IP Válido na Internet para que o backdoor consiga conectar.

   RESUMO DESTA CONEXÃO REVERSA:

 







Responda as questões abaixo para a conclusão da aula:

1-) Se tratando de conexão reversa, qual a afirmação verdadeira?
a) O computador que inicia a conexão é um servidor.
b) O computador que recebe a conexão possui o mesmo IP que o coputador cliente.
c) O Computador que inicia a conexão fica vulnerável ao computador que recebe a conexão.
d) O Computador que recebe a conexão fica vulnerável ao computador que inicia a conexão.
e) Nenhuma das alternativas.

2-) Por que a firewall pode ser lubridiado por um Backdoor de conexao reversa?
a) Porque o Backdoor não deixar nenhuma porta em escuta no computador alvo.
b) Porque o Backdoor de conexao reversa tem o código invisível para o antivírus.
c) Porque o IP conectado não está na lista do firewall.
d) Porque o DNS  conectado não está na lista do firewall.
e) Nenhuma das alternativas.

3-) Qual o melhor tipo de IP para quem irá fazer um ataque com conexão reversa.
a) IP Dinâmico
b) IP Local (Localhost)
c) IP fixo ou endereço DNS resolvendo em seu computador
d) O IP do atacante não tem importância
e) Nenhuma das alternativas.
