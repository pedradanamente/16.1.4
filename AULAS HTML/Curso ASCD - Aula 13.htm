TÚNEL DE DADOS
 

REVISÃO

Andre, neste tutorial vamos aprender sobre dados trafegando através de um túnel.

Antes vamos rever algumas coisas:
Para haver uma conexão entre dois computadores no mínimo deve existir um cliente e um servidor.
O servidor fica aguardando a conexão (modo escuta "Listing" )
O cliente é quem realiza a conexão. (modo "Conect")

Exemplo:



O servidor de web fica aguardando sua conexão.
 E seu computador conecta no servidor.



  TÚNELDE DADOS

"Tunelar" dados é fazer com que estes dados trafeguem através de um terceiro computador, que irá servir de "ligação" entre cliente e servidor.



Podemos utilizar um túnel para estudos de protocolos de comunicação, pois através do túnel você pode visualizar os comandos entre cliente e servidor.



CRIANDO UM TÚNEL (Exercício prático)

Vamos usar o Programa Insecuritynet 3.X para criar nosso túnel.

Primeiro vamos definir quem será o cliente, o túnel e o servidor:

Cliente Túnel Servidor
Nosso computador. Programa insecuritynet Servidor de FTP da Unicamp 


Vamos criar nosso túnel.

1-) Abra o programa e entre em:
Iniciar -> Defesa -> Port Tunel



2-) Vamos escolher como o túnel será conectado:
Nosso túnel deverá aguardar uma conexão e conectar no servidor da unicamp.
Então teremos que utilizar a opção "Listing -> Conect"

Vamos configurar o túnel para aguardar na porta 21 e redirecionar a conexão para o servidor "ftp.unicamp.br" na porta 21.
A porta 21 é a porta padrão para servidores FTP.



O túnel será aberto e ficará aguardando à conexão.



3-) Vamos criar a conexão.
Agora nosso computador será "virtualmente" o servidor de FTP da unicamp

Vamos conectar em nosso "SUPOSTO SERVIDOR FTP LOCAL"
Abra uma pasta qualquer e digite o seguinte endereço na barra: ftp://localhost



4-) Observe o TÚNEL (VISUALIZANDO OS DADOS)



Você pode visualizar toda a comunicação entre cliente e servidor.

Os dados em VERDE são dados de entrada (Nosso cliente)
Os dados em AZUL são dados de saída (Servidor Unicamp)

Conclusão

Quando você cria um Túnel deste tipo seu computador passa a ser também um servidor.
Se outro computador conectar no túnel, automaticamente será conectado ao seu redirecionamento.

Como deu para perceber o túnel é uma ótima ferramenta para estudos de protocolos e conexões.

Neste caso o túnel está em nosso computador, mas se o túnel estivesse em outro computador e você fosse induzido a usar o endereço do outro computador, Sua senha seria facilmente capturada.

 


EXEMPLO DE INTERCEPTAÇÃO DE DADOS
Definição de
CRACKER  		 Sujeito que quebra sistemas de segurança na intenção de obter proveito pessoal.
Possui um auto nível de conhecimento, porem sem ética, criminoso.

O protocolo POP não encripta nenhum dado.

Observe como um "Cracker" pode capturar uma senha de seu email:

1-) O "Cracker" faz ataques na sua rede para desestabilizar sua conexão ou a conexão de sua empresa.

2-) O cracker envia um falso email alegado que o endereço do servidor de email foi modificado.

3-) Inocentemente você configura o Outlook de seus computadores com o novo endereço.
O novo endereço, na verdade é o endereço do computador do "Cracker".
O computador do cracker na verdade é um tunel para o computador real.
Quando você conectar o outlook todos os dados estarão sendo interceptados.


Faça você mesmo a experiência.

Andre, caso você não tenha o Outlok habilitado em seu computador, entre em:
Iniciar -> Painel de Controle -> Adicionar e Remover Programas -> Adicionar e remover Componentes do Windows

Habilite o OUTLOOK




FALSA CONFIGURAÇÃO (No Computador alvo) INDUZIDA PELO CRACKER
Configure com estes endereços:
POP: localhost
SMTP: smtps.bol.com.br



No lugar do localhost o cracker iria usar seu IP ou um DNS parecido com o real.

Com um túnel de dados você pode visualizar os dados da conexão entre cliente e servidor, pois o computador túnel é uma ponte de ligação, assim você pode verificar a segurança de um protocolo de comunicação, neste exemplo vimos que o protocolo POP não oferece nenhum tipo de segurança, pois seus dados poderiam ser facilmente capturados.
